## Architecture Overview

This is a **React + TypeScript + Vite** field operations management app for Suburban Services (a cleaning services company). It uses **Supabase** as the backend database (no separate API server — all DB calls are direct via the Supabase JS client).

### User Roles & Auth

Four roles: `cleaner`, `manager`, `ops_manager`, `admin`. Auth is custom (not Supabase Auth) — passwords are hashed with bcryptjs and stored in the database. The `authService` handles login/registration by querying the appropriate DB table directly.

Session state is stored in `localStorage`:
- `userName` — cleaner/manager name (managed via `src/lib/identity.ts`)
- `userId`, `userType`, `managerId`, etc.

### Routing (`src/App.tsx`)

All routes are flat (no nested route groups). Role-based access is enforced at the page/component level rather than via route guards. Key routes:
- `/cleaner-dashboard` — cleaner's main view
- `/manager-dashboard`, `/ops-dashboard`, `/admin-dashboard` — all resolve to `ManagerDashboardPage`
- `/clock-in`, `/scanner` — QR scanning flows
- `/area-tasks` — task management
- `/analytics`, `/manager-activity`, `/bathroom-assist` — reporting
- `/qr-generator`, `/qr-library` — admin QR management

### Data Layer (`src/services/`)

Each service file is a collection of plain async functions (not classes) that call `supabase` directly:
- `authService.ts` — login, registration, user lookup
- `qrService.ts` — QR code generation, scanning, task logging
- `areasService.ts`, `areaTasksService.ts` — area/task CRUD
- `managerService.ts` — manager/cleaner relationships, customer linking
- `analyticsService.ts` — aggregated stats for dashboards
- `assistRequestService.ts` — bathroom assist request lifecycle
- `scheduleService.ts`, `calendarService.ts`, `opsVisitsService.ts` — scheduling

Database types are all defined in `src/services/supabase.ts`.

### QR Code Workflow

Central feature of the app. The flow: cleaner scans a QR → `QRScanner` component decodes it → calls `qrService` to validate and log the event → triggers `WorkflowManager` or `OpsWorkflowManager` for task selection.

QR types: `CLOCK_IN`, `CLOCK_OUT`, `AREA`, `TASK`, `FEEDBACK`. The `ClockOutValidator` handles the clock-out step within the area scan flow.

Offline draft persistence uses IndexedDB (`src/lib/offlineStore.ts`) to survive page reloads mid-workflow.

### Manager Scope (`src/lib/managerScope.ts`)

Hardcoded mapping of manager IDs/names to customer tokens. Managers only see data for their assigned customers. The `resolveManagerCustomerScope` function is called early in manager-facing pages to filter all subsequent queries.

### UI

- **shadcn/ui** components in `src/components/ui/` (do not edit these — they are auto-generated by the shadcn CLI)
- **Tailwind CSS** for all styling
- **TanStack Query** (`@tanstack/react-query`) for server state, with a single `QueryClient` instance in `App.tsx`
- **Recharts** for analytics charts (`src/components/analytics/`)
- `DashboardLayout` wraps all authenticated pages and renders a role-aware `Sidebar`
